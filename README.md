# GestureTalk

Мобильное Android-приложение для распознавания русского и американского жестового языка в реальном времени с использованием нейронных сетей.

## О проекте

GestureTalk — это дипломный проект, разработанный для помощи людям с нарушениями слуха в общении. Приложение использует камеру смартфона и модели машинного обучения для распознавания жестов и преобразования их в текст.

### Основные возможности

- **Распознавание жестов в реальном времени** — камера анализирует движения рук и выдает результат за несколько секунд
- **Две модели распознавания:**
  - **РЖЯ-1000** (русский жестовый язык) — модель MViT, обученная на датасете Slovo
  - **ASL** (американский жестовый язык) — модель TGCN, работающая с координатами ключевых точек
- **Детекция рук и тела** — MediaPipe отслеживает 21 точку руки и 33 точки тела
- **Визуализация ключевых точек** — можно видеть, как система "видит" ваши руки
- **AI-ассистент** — интеграция с GigaChat API для ответов на вопросы о жестовом языке
- **Справочник жестов** — обучающие материалы с видео и описаниями
- **Статистика и достижения** — отслеживание прогресса в изучении жестов
- **Синхронизация** — данные сохраняются в Firebase и синхронизируются между устройствами

## Как работает система

### Процесс распознавания

1. **Захват видео** — CameraX непрерывно получает кадры с камеры (30 FPS)
2. **Детекция рук** — MediaPipe находит руки в кадре и извлекает координаты 21 ключевой точки
3. **Определение движения** — система отслеживает изменение положения рук между кадрами
4. **Сегментация жестов** — алгоритм определяет начало и конец жеста

**Для модели РЖЯ-1000 (MViT):**
- Собирается последовательность кадров (сегмент)
- Добавляется pre-roll (5 кадров до движения) и post-roll (5 кадров после)
- Сегмент нормализуется до нужной длины (обычно 32 кадра)
- Кадры препроцессятся (resize, нормализация RGB)
- Модель MViT анализирует видеопоследовательность
- Выдается результат с вероятностью

**Для модели ASL (TGCN):**
- Собираются координаты ключевых точек рук
- Формируется граф связей между точками
- Временная последовательность графов подается в TGCN
- Модель анализирует динамику изменения графа
- Выдается распознанный жест

5. **Постобработка** — фильтрация по порогу уверенности, защита от повторов
6. **Вывод результата** — текст отображается на экране и сохраняется в историю

### Конвейерная архитектура (Hybrid Pipeline)

Система использует конвейерный подход для непрерывной работы:

```
Камера → MediaPipe → Сегментация → Очередь сегментов → ONNX Runtime → Результаты
   ↓                                      ↓
Продолжает работать            Пока модель думает,
                               камера собирает следующий жест
```

Это позволяет:
- Не терять жесты во время обработки предыдущего
- Поддерживать плавную работу камеры
- Обрабатывать несколько жестов подряд

### Алгоритм детекции начала/конца жеста

Используется комбинация метрик:
- **Движение рук** — вычисляется максимальное смещение ключевых точек между кадрами
- **EMA (экспоненциальное скользящее среднее)** — сглаживание шума
- **Пороги начала и конца** — MOTION_START (0.0075) и MOTION_STOP (0.0035)
- **Хвост (tail)** — еще 28 кадров после остановки движения для полной позы
- **Pre-roll** — последние 5 кадров перед началом движения для контекста

## Установка и запуск

### Требования

- Android Studio Arctic Fox или новее
- JDK 11
- Android SDK API 29+
- Git LFS (для загрузки моделей)
- Устройство или эмулятор с Android 10+
- Камера (на эмуляторе можно использовать виртуальную)

### Шаги установки

1. **Клонирование репозитория**

```bash
git clone https://github.com/AzimBulatov/GestureTalk.git
cd GestureTalk
```

2. **Установка Git LFS и загрузка моделей**

Git LFS нужен для больших файлов моделей (134 MB).

```bash
# Установка Git LFS (если еще не установлен)
# Windows: скачайте с https://git-lfs.github.com/
# Linux: sudo apt-get install git-lfs
# macOS: brew install git-lfs

# Инициализация LFS
git lfs install

# Загрузка моделей
git lfs pull
```

3. **Настройка Firebase** (опционально)

Если хотите использовать авторизацию и синхронизацию:

- Создайте проект в [Firebase Console](https://console.firebase.google.com/)
- Добавьте Android-приложение с package name `com.example.gesturetalk`
- Скачайте `google-services.json` и поместите в `app/`
- Включите Authentication (Email/Password) и Realtime Database

Если не нужна синхронизация, приложение будет работать локально.

4. **Открытие в Android Studio**

- Откройте Android Studio
- File → Open → выберите папку GestureTalk
- Дождитесь синхронизации Gradle (может занять несколько минут)

5. **Сборка и запуск**

- Подключите Android-устройство или запустите эмулятор
- Нажмите Run (зеленый треугольник) или Shift+F10
- Приложение установится и запустится

### Первый запуск

1. При первом запуске нужно создать аккаунт или войти
2. Приложение запросит разрешение на использование камеры — разрешите
3. Выберите модель в настройках (кнопка профиля → Настройки):
   - **mvit32-2.onnx** — для русского жестового языка
   - **asl1000.onnx** — для американского жестового языка
4. Вернитесь на главный экран и нажмите "Старт"
5. Покажите жест перед камерой

**Камера:**

- Автофокус (вкл/выкл)

**Визуализация:**
- Показ ключевых точек (landmarks)

## Производительность

**Slovo MViT:**
- Время инференса: ~2-3 секунды на Snapdragon 870
- Точность: зависит от качества выполнения жеста
- Поддерживает 1000 жестов

**ASL TGCN:**
- Время инференса: ~5 секунд (модель тяжелее)
- Работает только с координатами точек (меньше данных)
- Поддерживает 1000 жестов ASL

**Оптимизации:**
- Конвейерная обработка (не теряем жесты)
- Кэширование моделей
- Асинхронная работа через Kotlin Coroutines
- Умная сегментация (не обрабатываем пустые кадры)

## Известные ограничения

- Требуется хорошее освещение
- Руки должны быть в кадре полностью
- Фон не должен быть слишком загроможденным
- Работает на Android 10+ (API 29+)
- Модели занимают ~170 MB места
